Low-code and no-code are approaches to software development that aim to simplify
the process of building applications, making it accessible to non-developers or those
with limited coding experience.

### Flow of Application

In a low-code/no-code application, all functionalities are executed based on a configuration JSON retrieved from the backend. When the application loads for the first time, it calls an API to fetch this configuration JSON data. This configuration determines which site is loading and includes metadata, headers, footers, site menus, authentication menus, and admin menus configuration.

On the first app load, we call the configuration API for application settings. The configuration is divided into six main categories.

- [**Meta Config**](#meta) `(authMenu)` : It have Some meta information for seo and branding info.
- [**Site Config**](#site-menu) `(siteMenu)` : It have All Public/Private Web pages configuration JSON.
- [**Header Config**](#header) `(header)` : It have header configuration JSON.
- [**Footer Config**](#footer) `(header)` : It have footer pages configuration JSON.
- [**Auth Config**](#auth-menu) `(auth-menu)` : It have auth pages configuration JSON.
- [**Admin Config**](#admin-menu) `(adminMenu)` : It have Admin Pages/Dashboard pages configuration JSON.

if site open Non logged-in user they do not have access for Dashboard/Admin Pages. So their json have empty `adminMenu`. and when user logged-in then they open application then they have access to open `Dashboard/adminMenu`.

### Meta

The metadata encompasses site-specific details that vary from one site to another. This includes elements such as the site title, description, footer copyright text, logo, and additional relevant information. These details are essential for defining the site's identity and branding, ensuring each site has a unique and consistent presentation. For instance, the title and description are used for search engine optimization and browser tab labels, the footer copyright text provides legal and ownership information, and the logo represents the site's brand visually.

| Property         | Description                                                                          |
| ---------------- | :----------------------------------------------------------------------------------- |
| `title`          | The title of the site, which appears on the browser tab and is used for SEO purposes |
| `description`    | A brief summary of the site’s content, also used for SEO.                            |
| `copyright_text` | Legal and ownership information displayed in the footer.                             |
| `login_logo`     | A visual representation of the site's brand                                          |
| `sidebar_icon`   | A visual representation of the site's brand for sidebar                              |

### Site Menu

The site menu contains a list of public pages in a JSON format, accessible by both logged-in and non-logged-in users. The site menu is an array of objects, with each object including the following properties:

| Property       | Description                                  |
| -------------- | :------------------------------------------- |
| `templateName` | The name of the template used for the page.  |
| `label`        | The display name of the menu item.           |
| `url`          | The URL of the page.                         |
| `entity`       | The entity associated with the page.         |
| `icon`         | The icon representing the menu item.         |
| `id`           | A unique identifier for the menu item.       |
| `cache`        | Indicates whether the page should be cached. |
| `local`        | Specifies if the page is stored locally.     |
| `key`          | A key used for identifying the menu item.    |

These details collectively define the structure and behavior of the site menu, ensuring it is easily accessible and navigable for all users.

### Admin Menu

Admin menu contain list of pages of dashboard that have access to user. Admin menu is array of object which contain template name, label, url, entity, icon, id, cache, local, and key.

| Property   | Description                                                                                               |
| ---------- | :-------------------------------------------------------------------------------------------------------- |
| `template` | The name of the template used for the page                                                                |
| `label`    | The display name of the menu item                                                                         |
| `url`      | The URL of the page                                                                                       |
| `entity`   | The endpoint of API                                                                                       |
| `icon`     | The icon representing the menu item                                                                       |
| `id`       | A unique identifier for the menu item.                                                                    |
| `caches`   | Indicates whether the page should be cached.                                                              |
| `key`      | A key used for identifying the menu item and also used to highlight menu when open dashboard details page |
| `order`    | Order is used to manage order of display in menus                                                         |

### Auth Menu

Auth menu contain list of pages that can be used before login user like login, register verify user ec. Auth menu is array of object which contain template name, label, url, entity, icon, id, cache, local, and key.

| Property    | Description                                                                                                                                                                                                                          |
| ----------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `label`     | This label is Display Where we show list of page                                                                                                                                                                                     |
| `template`  | This is template name, and template name is used to Mapping template and Render match template.                                                                                                                                      |
| `url`       | This is page url                                                                                                                                                                                                                     |
| `id`        | id is used to uniquely identify menu in all menu array                                                                                                                                                                               |
| `caches`    | caches is used to store caches of api json data                                                                                                                                                                                      |
| `nextPage`  | This is next page url. After auth form submit success fully then we can redirect user to next page using                                                                                                                             |
| `param`     | if we want to send some data from one to another auth pages after form submit successful then param parameter will help them. We got name of form filed from param where we can collect info using name and pass into another pages. |
| `prevParam` | It have data of that is pass by previous pages.                                                                                                                                                                                      |
